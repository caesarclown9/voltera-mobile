Политика конфиденциальности

Версия 1.0.1 · Обновлено: 8 октября 2025

1. Введение

Voltera уважает вашу конфиденциальность и защищает персональные данные. Здесь описано, какие данные мы обрабатываем и зачем.

2. Какие данные мы обрабатываем

  • Аккаунт: email, телефон, имя (при наличии)
  • Баланс и транзакции (для пополнений и истории)
  • История зарядок
  • Геолокация — только с вашего разрешения (для поиска станций)
  • Технические данные устройства (модель, ОС, версия приложения)
  • Токены пуш‑уведомлений (по вашему выбору)

3. Зачем это нужно

  • Предоставление услуг: вход, баланс, запуск/остановка зарядки
  • Уведомления о статусе зарядки и системные сообщения
  • Улучшение качества сервиса и безопасность
  • Соблюдение закона

4. Партнёры‑обработчики

  • Supabase (Auth/DB/Realtime, ЕС)
  • Платёжные провайдеры (для пополнений)
  • Картографический провайдер (карта без передачи персональных данных)

5. Безопасность и хранение

Передача по HTTPS/TLS, хранение паролей в виде хэшей, ограниченный доступ. История платежей хранится согласно закону, прочие данные — пока аккаунт активен (или удаляются по запросу).

6. Ваши права

Доступ, исправление, удаление, ограничение обработки, переносимость, отзыв согласия. Связь: support@voltera.kg

7. Разрешения приложения

  • INTERNET — работа с сервером
  • ACCESS_FINE_LOCATION / COARSE_LOCATION — поиск ближайших станций (опционально)
  • CAMERA — сканирование QR (опционально)
  • POST_NOTIFICATIONS (Android 13+) — уведомления (опционально)

8. Контакты

Email: support@voltera.kg
Адрес: Кыргызская Республика, г. Бишкек

9. Категории данных и цели (Data Safety)

Мы указываем категории данных, которые могут обрабатываться, и цели обработки:

  • Персональные данные: email, телефон, имя — для регистрации, входа, поддержки.
  • Финансовые данные: баланс, платежи — для пополнений и расчётов.
  • Местоположение: точное/приблизительное (только при согласии) — для поиска ближайших станций.
  • Идентификаторы: токены пуш‑уведомлений, идентификаторы сессии — для уведомлений и безопасности.
  • Технические данные: модель устройства, ОС, версия приложения — для диагностики и улучшений.

Мы не продаём данные и не используем их для рекламы поведенческого таргетинга.

10. Передача третьим сторонам

  • Supabase (ЕС): аутентификация, база данных, realtime. Доступ по принципу минимально необходимого.
  • Платёжные провайдеры (O!Dengi, банк): обработка пополнений и статусов платежей.
  • Карты (провайдер карт): тайлы/SDK для отображения карты; персональные данные не передаются.

Все передачи выполняются по HTTPS/TLS. Перечень и цели соответствуют разделу 9.

11. Хранение, шифрование и сроки

  • Передача — только по HTTPS/TLS.
  • Пароли — только в виде хэшей, к ним нет прямого доступа.
  • Сроки хранения: платёжные записи — согласно закону; прочие данные — пока аккаунт активен или до удаления по запросу.
  • Доступ ограничен по ролям. Применяются политики Row Level Security для БД.

12. Управление данными и удаление

  • Доступ/экспорт: запросите копию данных через поддержку.
  • Исправление: обновите профиль или обратитесь в поддержку.
  • Отзыв согласий: отключите геолокацию/уведомления в настройках устройства и приложения.
  • Удаление аккаунта и данных (in‑app): Профиль → Настройки → Удалить аккаунт. Невозвратная операция. Часть платёжных записей может сохраняться по требованиям закона.

13. Дети

Сервис не предназначен для детей. Мы не привлекаем и не таргетируем пользователей младше действующего порога возраста по местному праву.

14. Изменения этой политики

Мы можем обновлять документ при изменении требований или функционала. Дата версии всегда указана вверху. Существенные изменения будут отражены в приложении.
